# Harness Routing Intent

## Purpose

Map selected harnesses and CLIs to the providers/models they should use.

## Required visible language

- routing
- Hermes
- Copilot

## Inputs and defaults

- Routing selection/default

## Required actions

- Configure provider authentication once routing is valid

## Blocking behavior

- Routing must be present for selected harness and CLI.

## Setup payload expectations

- Routing data maps selected harness/CLI to selected provider/model.

## Visual expectations

- Routing summary should be visible and understandable.
