# Models Intent

## Purpose

Confirm default models and available model choices for the selected provider.

## Required visible language

- model
- GitHub Copilot

## Inputs and defaults

- Model selection/default

## Required actions

- Choose harness routing after model defaults are valid

## Blocking behavior

- A default model must be visible before continuing.

## Setup payload expectations

- Selected/default models are included in setup payload model data.

## Visual expectations

- Model choices are visible enough for review without opening developer tools.
