# Harnesses Intent

## Purpose

Choose the agent harness that will run CTO Desktop automation.

## Required visible language

- Hermes
- harness

## Inputs and defaults

- Harness selection

## Required actions

- Select Hermes for the current local CTO lane
- Continue to dynamic workflows after a compatible harness is selected

## Blocking behavior

- A harness must be selected before continuing.

## Setup payload expectations

- Selected harness is included in the setup payload.

## Visual expectations

- Selected harness is visibly distinct from unselected choices.
